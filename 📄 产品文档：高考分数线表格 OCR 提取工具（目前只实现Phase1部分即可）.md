# 📄 产品文档：高考分数线表格 OCR 提取工具（目前只实现Phase1部分即可）

## 一、产品定位

一个面向“高考分数线图片”的**结构化提取工具**，支持：

- 图片 → 表格数据
- 自动识别 + 人工校对
- 导出 Excel / CSV

👉 核心目标不是“100% OCR”，而是：
**用最小人工成本，把图片变成可用数据**

------

## 二、目标用户

- 整理高考数据的学生 / 自媒体
- 做志愿填报分析的人
- 爬虫 / 数据收集人员

------

## 三、核心功能

### 1️⃣ 图片上传

- 支持 jpg / png
- 支持拖拽上传
- 支持多图（可选）

------

### 2️⃣ OCR识别（核心）

使用 PaddleOCR 表格识别能力：

输入：

- 图片

输出：

```
{
  "headers": ["年份", "省份", "科类", "批次", "分数"],
  "rows": [
    ["2023", "河南", "理科", "一本", "514"],
    ...
  ]
}
```

------

### 3️⃣ 表格编辑（重点）

👉 这是产品价值核心（比 OCR 还重要）

功能：

- 可编辑表格（类似 Excel）
- 单元格点击修改
- 支持新增 / 删除行
- 支持列名修改

------

### 4️⃣ 智能校验（加分项 ⭐）

自动标记异常数据：

- 年份不是 4 位
- 分数不是数字
- 分数超范围（比如 >750）
- 空值

UI表现：

- 高亮（红 / 黄）
- hover 提示错误原因

------

### 5️⃣ 导出功能

支持：

- CSV
- Excel（.xlsx）

------

### 6️⃣ 原图对照（很重要）

👉 左图右表结构：

- 左侧：原始图片
- 右侧：识别表格
- 点击单元格 → 高亮图片区域（进阶）

------

## 四、产品流程

```
上传图片
   ↓
OCR识别（后端）
   ↓
返回结构化数据
   ↓
前端渲染表格
   ↓
用户修改 / 校验
   ↓
导出数据
```

------

```

  "rows": [...]
```

响应：

- 文件流（csv / xlsx）

## 八、OCR 服务设计（重点）

### Python FastAPI 示例结构：

```
@app.post("/ocr")
async def ocr(file: UploadFile):
    image = read_image(file)

    result = ocr_engine.ocr(image)

    table = parse_to_table(result)

    return table
```

------

### PaddleOCR 使用建议

使用：

```
PaddleOCR(
    use_angle_cls=True,
    lang="ch"
)
```

👉 如果做表格识别，建议：

- 使用 PaddleX table pipeline（更稳）

## 十一、难点分析（提前帮你踩坑）

### ⚠️ 1. 表格错位

👉 最常见问题

解决：

- 不直接用 OCR 文本
- 用“单元格结构识别”

------

### ⚠️ 2. 图片质量差

👉 模糊 / 倾斜

解决：

- 预处理（OpenCV）
- 自动拉正

------

### ⚠️ 3. 数字识别错误

👉 518 → 51B

解决：

- 后处理校验（非常关键）
